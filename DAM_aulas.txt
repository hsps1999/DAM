ANDROID TUTORIALS:

- A compilação de código Android gera ficheiros no formato Dalvik Executable com a extensão .dex;
- Os ficheiros, juntamente com os recursos e os arquivos de manifesto, são agrupados em ficheiros .apk;
- Um ficheiro .apk é uma app Android pronta a ser instalada;
- Existem quatro (mais um Layout) componentes principais numa aplicação Android:
  - ACTIVITIES: (camada organizacional) Suporta uma tela para interação do usuário. Uma app pode ter várias atividades. Uma atividade tem um Layout Container para agrupar os elementos da UI;
  - LAYOUT CONTAINERS: (camada de apresentação) É um contentor que contém elementos de UI (User Interface) numa tela. Agrupa elementos de design chamados Widgets;
  - SERVICES: (camada de processamento em segundo plano) Componente que executa em background, fornecendo serviços para as atividades;
  - BROADCAST RECEIVERS: (camada de notificação) Componente que permite que as atividades recebam notificações do sistema e reajam às mesmas;
  - CONTENT PROVIDERS: (camada de armazenamento) Componente que armazena e fornece dados para as atividades.
- Classes auxiliares importantes:
  - ANDROID FRAGMENTS: Partes reutilizãveis da UI;
  - ANDROID INTENTS: Permitem comunicação entre os componentes.

- Layout Containers são construídos com base na classe ViewGroup e os Widgets com base na classe View.
- Para gráficos Android como imagens ou animações, é usado o termo Drawables.
- Android Virtual Devices (VDs/ AVDs)
- Android Debug Bridge (ADBs)
- Telemóvel > Opções de programador > Depuração USB

- CREATE APP:
  - File > New > New Project > Empty Views Activity

  - quando solicitado, digitar código de pareamento;

TPC:
Unit testing é uma prática essencial para garantir a qualidade do código. Envolve a criação de testes automatizads para verificar se partes específicas do seu código (unidades) funcionam corrretamente. 
-Configurar projeto:
  - Especificar JUnit 5 como framework de teste ao criar projeto;
- Adicionar dependências:
  - No arquivo build.gradle (ou build.gradle.kts), adicionar depndência para trabalhar com testes usando kotlin.teste e JUnit:
 dependencies {
    // Outras dependências...
    testImplementation 'org.jetbrains.kotlin:kotlin-test'
 }
- Criar um teste:
  - Abrir arquivo main.kt em src/main/kotlin;
  - Criar classe camada Sample com uma função (ex:sum(a:int, b:int);
  - Testar:
 import kotlin.test.Test
 import kotlin.test.assertEquals

 internal class SampleTest {
    private val testSample: Sample = Sample()

    @Test
    fun testSum() {
        val expected = 42
        assertEquals(expected, testSample.sum(40, 2))
    }
 }
- Executar teste
